프로젝트에서 경로를 효율적으로 관리할수 있는 경로 관리자 만들어줘.
시간이 많이 소요되는 작업은 비동기로 처리를 우선하기 위한 스래드 매니저를 설계해줘. 다형형을 적용하고 베이스 클래스를 상속 받은 워크 스래드와 스래드 매니저 클래스로 만들어줘. 스래드는 비동기로 설계해줘.
시퀀스 설계에서 사용할 타이머 관리자 설계해줘. 다형형을 적용하고 베이스 클래스를 상속 받은 타이머 클래스로 만들어줘. 타이머 동작은 고속, 정밀 타이머, loop 타이머, off 타이머, on 타이머로 우선 설계해줘.
각 워커 스래드에서 사용할 수 있도록 스래드 세이프하게 설계해줘.
프로젝트에서 경로를 효율적으로 관리할수 있는 파일 관리자 만들어줘.
프로젝트에서 통신를 효율적으로 관리할수 있는 통신 관리자 만들어줘. 다형형을 적용하고 베이스 클래스를 상속 받은 통신 클래스로 만들어줘. 통신은 비동기 소켓 통신 (서버, 클라이언트), 시리얼 통신으로 설계해줘.
프로젝트에서 레시피 효율적으로 관리할수 있는 레시피 관리자 만들어줘. 다형형을 적용하고 베이스 클래스를 상속 받은 레시피 클래스로 만들어줘. 레시피는 패턴 매칭용 모델, 설정 파일을 관리할수 있게 설계해줘.
프로그램 시작 시 이전 마지막 레시피를 자동 로딩하게하고, 새로운 레시피를 생성, 기존 레시피 복사, 수정, 삭제 기능으로 설계해줘.
프로젝트에서 환경설정을 효율적으로 관리할수 있는 설정 관리자 만들어줘. 다형형을 적용하고 베이스 클래스를 상속 받은 설정 클래스로 만들어줘. 설정에서는 클라이언트 프로그램에서 다양하게 설정할 수 있게 설계해줘.
설정 관리자는 모듈별 설정을 관리 할수 있도록 해줘. 예를 들어 통신 모듈은 소켓 통신, 시리얼 통신 관련 설정, 파일 모듈은 파일 삭제 주기 등. 

BGA ball 검사를 위한 제품 정보 설정이 필요해요. 프로젝트에서 제품 정보를 효율적으로 관리할수 있는 제품 관리자 만들어줘.
다형형을 적용하고 베이스 클래스를 상속 받은 제품 클래스로 만들어줘. 
제품 정보는 substrate 이고 윈도우 갯수, 각윈도우 가로 세로 유니트 배열, 유니트 피치, 윈도우 피치를 설정할 수 있도록 설계해줘.
유니트 정보는 ball arrary (x, y), ball size, ball pitch 설정도 설계해줘.

프로젝트 관리를 위해 Vision 관련 내용은 라이브러리 프로젝트로 생성하여 관리하게 설계해줘.
예를 들어 Lib_Image, Lib_ImageGrabber, Lib_Insp_D2D, Lib_Insp_Base, Lib_Blob, Lib_Matching, Lib_Mopology, Lib_PreProcess.

설계하신 reduce domain → 이진화 → 모폴로지 → 연결(윤곽) → 블롭 순서는 현재 OpenCV 파이프라인과 동일합니다.
빨간 원은 현재는 상태별 색상이라, “검출만 전부 빨강”이 필요하면 표시 옵션을 추가하는 것이 좋습니다.
효과 대비 비용이 좋은 개선은 ROI Mat 뷰로 복사 줄이기, 원형 모폴로지·반복 횟수 옵션, 오버레이 색/안티앨리어싱입니다.
원하시면 다음 단계로 (A) 검출 원만 전부 빨강 + 설정으로 토글 또는 (B) ROI를 Mat.Roi로 바꾸는 리팩터 중 어디부터 코드에 반영할지 정해 주시면 그에 맞춰 수정안을 제안하겠습니다.
